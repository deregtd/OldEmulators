News:

10/31/99:

  • Yes, I'm still alive. NES4PC has not been dropped. I finally got a freaking working directdraw core, that works in both windowed and full-screen, so I'll be able to add that soon. It will, however, require DirectX 7.0, so start up on your downloads of that. ;-) I've been working on nes4pc, and frankly, I can't figure out what's wrong with the damn thing. I've checked every part of it a million times over and I just can't find out why certain things don't work. So I've decided to just focus on features like sound, game genie, save states, and video. I've located the problem in the sound core and am working on fixing it by letting directsound do the mixing instead of having nes4pc do it. I am also about to add directinput support, so the problem of suddenly losing control will disappear.
4/15/99:
  • Well, today I've been given the last straw to break my will to continue hacking together the current incarnation. I don't believe that with my current emulation method that I will be able to implement correct SNSS save states... My emulation method is also flawed in many other ways, a few of which being that the way I do name/attribute tables is just wrong, and that my cpu core, while easy to look at, is totally inefficient and makes converting routines to assembly near impossible. It is the combination of these and a million other problems with NES4PC that has made me decide to simply release v0.35 as is, without directdraw (but with a partial directdraw core included but not used) and with only-works-on-NES4PC SNSS support (aka don't try it on other people's computers). This weekend I will put together the final release package for v0.35 and then take as long as I need to rewrite my CPU core, Video core, GUI, mappers implementation, and a number of other things. The release will include a limited source code, meaning that I won't include _EVERYTHING_, but I'll include enough that someone can easily use it for reference. I originally planned to release the full source code in hopes that someone could help me fix some of the bugs, but I have come to realize that the only real fix is a rewrite, and so I will simply release enough to use as reference, but hopefully discouraging skript kiddies from replacing all text "NES4PC" with "LEETNES" or something and then releasing it as a "new emulator".
  • Don't worry, this doesn't mean I'm going to stop development of NES4PC. I simply want to have all the time I need to rewrite everything instead of keeping myself to a schedule and then constantly pushing it back. I will continue to put up notices on the page regarding my progress on the rewrites as I complete a major step. This is simply a major turning point for NES4PC. The new cpu core will be as accuate as current knowledge can make it, and as fast as my ASM skills can make it as well. The debugger will be as good or possibly better than the old one. Game Genie codes will work. The graphics will be done with DirectDraw and will be easily configurable for either windowed mode or full screen, and will be stretchable in windowed mode. Mapper support will be fixed, expanded, and optimized. Controllers will be done via DirectInput. However this all takes time, and so I can't say how long it will take. 2 to 3 months for it all would be a good guess. I may release something along the way to show what's been finished as well, so don't lose all hope!
4/13/99:
  • Well thanks to bpoint the DirectDraw core is fixed and moving right along. 16-bit color and 24-bit color will be easy to add. The hard part will be 8-bit color. In windowed mode, you cannot use directdraw to set a palette, and if you try to use the Windows API calls for manipulating palettes, then Windows GPFs, Blue Screns, then hangs. So I'm probably going to use my GDI video core for 8-bit color and use DirectDraw for 16-bit, 24-bit, and 32-bit color. Come to think of it, that would simplify my graphics core tremendously so I think that's what I'll do!
  • Unfortunately, we in Nesdev are still working on finalizing SNSS, but it is very close to completion. I'll tell you that there are at least two other NES emulators that have added SNSS and that are working with me on finalizing it; Nofredo and LoopyNES. We want to wait until we have a good finished SNSS product before we all release new versions of our emulators.

3/8/99:
  • Well, I've been working on a Pong homebrewn ROM that was started by Johannes. As I work on the ROM more and more, I find that some things are just wierd about my emu or something. =)

    For instance, what I have right now runs perfectly on NES4PC. On the latest loopynes beta, it runs without the scores and with a few graphics glitches at the bottom of the screen. On Nesticle, the graphics are entirely missing and you can just hear the sound. (The sound works right on all 3)

    So I've decided to make NES4PC an open source project, in the hopes that it will help people learn from it, and in the hopes that someone can help me make it the best emulator that it can be.

    All I ask by doing this is that anyone who uses my source either for reference gives me credit in their documentation, or anyone who uses blocks of the code notifies me via email what they're using. I just want to make sure that noone's ripping off my hours and hours of work...

    As soon as I get alpha 3 fixed (there are a number of bugs that I'm working on from new additions to the code), I will release it in the form of binaries and in the form of the full source code.

    The source code is made in Borland C++ Builder v1.0, using the DirectX 6.1 SDK for include files/lib files, although the DirectX 6.0 includes/libs work too.

    If someone wants to port NES4PC, give it a shot and notify me via email so I can put up a link and add full info on your project.

    If you find any bugs in the code, notify me via email as soon as you possibly can. =) If you successfully add a mapper to the source code, tell me and I'll add it to the source code and give you full credit for it.

    Lastly, I don't want anyone to, like, continue my work and just add on to my source code and start a new emulator with it.

    I will continue the emulator same as before, I am just hoping that this will benefit both NES4PC and the community.
